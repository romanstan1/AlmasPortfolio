Housing Bubble

BUILD: AngularJS, D3.js, HTML, PostgreSQL, Ruby on Rails, SCSS.

Housing Bubble was created with the primary aim to learn and practice the D3.js library. The application visualises property data retrieved from the Zoopla API. Houses are represented as SVG bubbles of different sizes and colours relative to the their listed value. Users can search for any location in the UK by Street name, Post code, Town, City or Area. The user can log in, add and remove houses to and from their housing portfolio and then view the house listed on Zoopla by clicking on the houses grey info box. Users can also sort houses by the number of Bedrooms or Bathrooms. There is a colour key that pops up when the footer is hovered over.

Snake

BUILD: CSS, JavaScript, jQuery, HTML.

This project was a modern remake of the classic Snake game. It includes an opening sequence that spells out the word 'SNAKE' with the snake itself. This version of Snake has four game options/levels which are: 1. 'Snake 2' with hollow walls, 2. 'Classic Snake' with solid walls, 3. 'Some walls' in the middle of the board and on the edge, 4. 'Bombs' that randomly appear on the game board. You can toggle through the levels using the the X and Z keys (only when the game is paused).

Lazy Recipes

BUILD: AWS, Express, HTML, JavaScript, MongoDB, Node.js, SCSS.

The appropriately titled “Lazy Recipes” is an application that finds recipes for users. The concept is simple - a user can upload images of food (or browse existing images) and then view recipes that can be cooked using the ingredients that are in that image. It uses an impressive image recognition API from Clarafai to achieve this along with a recipe finder API from Food2Fork. The design of the website is clean and rounded - this was to create a soft and inviting impression to the site, whilst allowing the content to be easily viewed and appreciated.

Bet Bucket

BUILD: AngularJS, ChartsJS, Express, HTML, MongoDB, Node.js, SCSS.

Bet Bucket is an application that can track and visualise real-time sports odds using multiple repeated requests to the Betfair API. Users can build several accumulators bets, each containing multiple tracked bets. Bets are selected through a 4-stage selection process, that requires users to delve into Betfair's in-play events and select: 1. Event-Type (eg. Soccer), 2. Event Name (eg. Real Madrid Vs Barcelona), 3. Market Name (eg. Match Odds) 4. Runner Name (eg. Barcelona). Once added to the users chosen accumulator, all events are tracked and visualised by the graph at the top of the page. Users can also enter a stake to see the returned amount of £'s should they place a bet.
